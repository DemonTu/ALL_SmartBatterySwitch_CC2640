\subsection{L\+C\+D\+Dogm1286.\+h File Reference}
\label{_l_c_d_dogm1286_8h}\index{L\+C\+D\+Dogm1286.\+h@{L\+C\+D\+Dogm1286.\+h}}


\subsubsection{Detailed Description}
L\+C\+D driver implementation for a D\+O\+G\+M128\+W-\/6 L\+C\+D display. 

============================================================================

\subsubsection*{Driver include}

The L\+C\+D header file should be included in an application as follows\+: 
\begin{DoxyCode}
\textcolor{preprocessor}{#include <ti/drivers/lcd/LCDDogm1286.h>}
\end{DoxyCode}


\subsubsection*{Overview}

This L\+C\+D driver implementation is designed to operate on a D\+O\+G\+M128\+W-\/6 L\+C\+D display. This display is the one used on the Smart\+R\+F06 E\+B (Evaluation Board). The L\+C\+D display can be written to by first writing the data to display to an internal buffer, and then sending the buffer content to the L\+C\+D display. The L\+C\+D driver will make use of the S\+P\+I driver for updating the display. The driver contains different functions for both modifying the internal buffer and for updating the display with the buffer content.

The D\+O\+G\+M128\+W-\/6 L\+C\+D display contains 128 pixel columns and 64 pixel rows. The (x,y) coordinate system is as described below\+: \begin{DoxyVerb}*
*       X ----->
*   Y   +---------------------------------------------+
*   |   |(0,0)              PAGE 0             (127,0)|
*   |   |                   PAGE 1                    |
*   v   |                   PAGE 2                    |
*       |                   PAGE 3                    |
*       |                   PAGE 4                    |
*       |                   PAGE 5                    |
*       |                   PAGE 6                    |
*       |(0,63)             PAGE 7            (127,63)|
*       +---------------------------------------------+
*  \end{DoxyVerb}


This L\+C\+D driver uses a 5x7 font, thus each character takes up 5 pixels width and 7 pixels height. Each line of text will take up one page of the display. The difference between the defined variables \hyperlink{_l_c_d_dogm1286_8h_a4fad5c4c39b66358c5ab60f53d26ac86}{L\+C\+D\+\_\+\+C\+H\+A\+R\+\_\+\+W\+I\+D\+T\+H} and \hyperlink{_l_c_d_dogm1286_8h_a311c85b23f68d1ae7b4c242c9fa14159}{L\+C\+D\+\_\+\+F\+O\+N\+T\+\_\+\+W\+I\+D\+T\+H} equals the character spacing on the L\+C\+D display.

\subsubsection*{General Behavior}

Before using L\+C\+D\+:
\begin{DoxyItemize}
\item In order to use the D\+O\+G\+M128\+W-\/6 L\+C\+D display on the Smart\+R\+F06 E\+B, the application needs to first drive the Board\+\_\+3\+V3\+\_\+\+E\+N pin high to enable the 3.\+3\+V domain on the board.
\item The L\+C\+D display is configured by calling \hyperlink{_l_c_d_dogm1286_8h_a7eda62cc528319af943746da6b5d7688}{L\+C\+D\+\_\+open()}. This function will in turn open the S\+P\+I driver and allocate the 2 additional control signals (through the P\+I\+N driver) required to operate the L\+C\+D.
\end{DoxyItemize}

In total the following pins are required to operate the L\+C\+D on the Smart\+R\+F06\+E\+B\+: \begin{DoxyVerb}*  Board_SPI0_MOSI     (Serial peripheral interface, Master-Out-Slave-In, controlled by LCD driver)
*  Board_SPI0_MISO     (Serial peripheral interface, Master-In-Slave-Out, controlled by LCD driver)
*  Board_SPI0_CLK      (Serial peripheral interface, Clock, controlled by LCD driver)
*  Board_LCD_CSN       (Serial peripheral interface, Chip select, controlled by LCD driver)
*  Board_LCD_RST       (LCD reset signal, active low, controlled by LCD driver)
*  Board_LCD_MODE      (LCD mode signal, controlled by LCD driver)
*  Board_3V3_EN        (Enable the 3.3V domain. NOTE: Must be set by application, NOT controlled by LCD driver)
*  \end{DoxyVerb}


The following S\+P\+I parameters will be fixed by the L\+C\+D driver\+:
\begin{DoxyItemize}
\item transfer\+Mode = \hyperlink{_s_p_i_8h_ab9ea76c6529d6076eee5e1c4a5a92c6fa0dfb2358e008316426895e7237c398e8}{S\+P\+I\+\_\+\+M\+O\+D\+E\+\_\+\+B\+L\+O\+C\+K\+I\+N\+G}
\item transfer\+Timeout = B\+I\+O\+S\+\_\+\+W\+A\+I\+T\+\_\+\+F\+O\+R\+E\+V\+E\+R
\item mode = \hyperlink{_s_p_i_8h_a60a7e3d74577b38aa79ea6983362f942a84379dc90398ca075038c8d5ee465f6a}{S\+P\+I\+\_\+\+M\+A\+S\+T\+E\+R}
\item data\+Size = 8
\end{DoxyItemize}

The A\+P\+Is in this driver serve as an interface to a typical T\+I-\/\+R\+T\+O\+S application. The specific peripheral implementations are responsible to create all the S\+Y\+S/\+B\+I\+O\+S specific primitives to allow for thread-\/safe operation.

After L\+C\+D operation has ended\+:
\begin{DoxyItemize}
\item Release system dependencies for L\+C\+D by calling \hyperlink{_l_c_d_dogm1286_8h_a9a6a62dcb23c554201420e7e555e6ecc}{L\+C\+D\+\_\+close()}.
\end{DoxyItemize}

\subsubsection*{Error handling}

Please refer to the S\+P\+I driver documentation for information about error handling during a S\+P\+I transaction.

If an error occurs when modifying the internal buffers, the functions are return without completing the buffer modification.

\subsubsection*{Power Management}

The T\+I-\/\+R\+T\+O\+S power management framework will try to put the device into the most power efficient mode whenever possible. Please see the technical reference manual for further details on each power mode.

The L\+C\+D driver makes use of the S\+P\+I\+C\+C26\+X\+X\+D\+M\+A driver, which is setting a power constraint during transfers to keep the device out of standby. I.\+e. device will enter idle mode when no tasks are active. When the transfer has finished, the power constraint is released.

\subsubsection*{L\+C\+D Details}

\paragraph*{Allocating space to internal buffers}

The L\+C\+D driver joined (at link time) to an array of \hyperlink{struct_l_c_d___buffer}{L\+C\+D\+\_\+\+Buffer} structures named {\itshape lcd\+Buffers}. {\itshape lcd\+Buffers} is implemented in the application with each entry being an instance of a \hyperlink{struct_l_c_d___buffer}{L\+C\+D\+\_\+\+Buffer}. The number of buffers availible must be passed as an argument to \hyperlink{_l_c_d_dogm1286_8h_a7eda62cc528319af943746da6b5d7688}{L\+C\+D\+\_\+open()}. Each entry in {\itshape lcd\+Buffers} contains a\+:
\begin{DoxyItemize}
\item (Char $\ast$) which points to a buffer with data to be printed on the L\+C\+D
\item (unsigned int) which is the size of the buffer
\item (Semaphore\+\_\+\+Struct) which is a semaphore to be associated with the buffer. The Semaphores will be constructed by the L\+C\+D driver when it is being opened, and is being used to block task execution while the associated buffer is in use.
\end{DoxyItemize}

\paragraph*{Modifying internal buffers and updating L\+C\+D display}

Before the L\+C\+D screen can be filled, the data to display must be written to the internal buffers. All functions called L\+C\+D\+\_\+bufferxxxx are only modifying the internal buffers. To send the buffer content to the L\+C\+D display, one must use the functions \hyperlink{_l_c_d_dogm1286_8h_a6974d1de55526b98c6af02c90fd34185}{L\+C\+D\+\_\+update()} or \hyperlink{_l_c_d_dogm1286_8h_abd9b38254fab63ff90f3b4b93d4c8d8b}{L\+C\+D\+\_\+update\+Part()}. The only exception from this is the function \hyperlink{_l_c_d_dogm1286_8h_a1208372099f0929bbbd1a5ec8621afce}{L\+C\+D\+\_\+write\+Line()}, which both modifies a buffer and send the content to the L\+C\+D display.

\subparagraph*{Configuring the L\+C\+D display}


\begin{DoxyCode}
LCD_Handle lcdHandle;
LCD_Params lcdParams;
PIN_State  pinState;
PIN_Handle hPins;

\textcolor{comment}{// This application supports two LCD buffers}
Char lcdBuffer0[LCD_BYTES] = \{0\};
Char lcdBuffer1[LCD_BYTES] = \{0\};

\textcolor{comment}{// Populate LCD\_Buffer structure with buffer pointers and buffer sizes.}
\textcolor{comment}{// The Semaphore structure will be constructed when opening the LCD.}
LCD_Buffer lcdBuffers[] = \{
    \{lcdBuffer0, LCD_BYTES, NULL\},
    \{lcdBuffer1, LCD_BYTES, NULL\},
\};

\textcolor{comment}{// Application pin table.}
\textcolor{comment}{// Drive Board\_3V3\_EN high to enble 3.3V domain on SRF06EB}
PIN_Config pinList[] = \{
     Board\_3V3\_EN     | PIN_GPIO_OUTPUT_EN | PIN_GPIO_HIGH  | PIN_PUSHPULL,
     PIN_TERMINATE
\};

\textcolor{comment}{// Open PIN driver}
hPins = PIN_open(&pinState, pinList);

\textcolor{comment}{// Initialize LCD parameters}
LCD_Params_init(&lcdParams);

\textcolor{comment}{// Optionally override default bit rate}
lcdParams.spiBitRate = someNewSpiBitRate;

\textcolor{comment}{// Open LCD driver, using 2 buffers}
lcdHandle = LCD_open(&lcdBuffers[0], 2, &lcdParams);

\textcolor{comment}{// Make sure it was successful}
\textcolor{keywordflow}{if} (!lcdHandle)
\{
    System\_abort(\textcolor{stringliteral}{"Error initializing LCD\(\backslash\)n"});
\}
\end{DoxyCode}
